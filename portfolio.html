Portfolio Website Project

Overview:

I created a portfolio website to showcase my design skills to employers and clients who would like to get to know me better. This website was designed and built with HTML, CSS, and Javascript.

Purpose:

The purpose of the project was to learn how to build and design a website for anyone interested in my designs or is looking to hire a designer. This frontend project was part of my CareerFoundry course.

Role & Duration:

4 months

  • UI/UX Designer
  • Frontend Developer

Tools and Languages:

  • Atom
  • Adobe Illustrator
  • HTML
  • CSS
  • Javascript

Objective:

Problem -  I was looking to create a portfolio website that would allow my designs to be the main focus. This website provides a way for anyone interested in my designs to view my projects, resume, and learn about me as a person and designer in one place.

Solution - I aimed for my website to be minimal and easy to use. Creating a portfolio website with a minimal design would make it easier for employers and clients to navigate through my website to get to know me and my design abilities.

Process and Solutions:

I learned how to use HTML and CSS to structure my website and design a layout for my portfolio. The website design started with a mobile-first approach. At first, the home page had a basic layout with a project image on the left and a text description on the right.

When I started to learn more advanced CSS, I created an animation of an image that was created by someone on Freepik. I chose to animate an image of Austin, Texas to hint at where I reside.

When I started to learn Javascript I was able to make the website more responsive.

I implemented CSS grids to organize my project images in columns and rows on the home page to create a minimal design. Users would be able to read a short description of the project when they hovered over a project image. It reduced clutter and allowed my projects to be the main focus.

I learned to make better use of space on my project pages by creating thumbnail images that would show up in a lightbox where users could better preview images that were part of my design process.

An active button was set in the navigation to make it easier for people to know where they are on the website.

User testing allowed me to find issues and ways to improve the website. I utilized the feedback to provide a better user experience by taking into consideration user behavior and the many ways users could be viewing my portfolio.

Usability Report:

Participants found that my website is clean and showed how creative I am. They felt it was easy to use and understand.

  • Problem
  • Users can only click View Project button
  • Chopped text and unnatural gap on medium screen size
  • User wants easier access to email
  • About is an unclear tab name
  • Copyright date was incorrect
  • Solution
  • Make project images clickable when hovered
  • Fix floating image in medium size screen
  • Add email at bottom of page
  • Change About to About Me
  • Update copyright year to be current year

Code quality and cross-browser testing allowed me to ensure that my website was scaled correctly and was not throwing errors. Color accessibility was checked and all colors used on the website were updated to be AA compliant. The functionality of my website was tested on Google Chrome, Apple’s Safari, Microsoft Edge, and Mozilla Firefox. I updated some of my code to be more compatible with more browsers to allow my website to run smoothly and provide an improved user experience.

Cross-Browser Issues:

  • Problem
  • Unnatural text gap caused by floating image on medium screens for all browsers
  • Project text is same color as links on mobile devices
  • Lightbox image is cut off when user clicks some images
  • Copyright text and email on different lines on mobile
  • Solution
  • Decreased max-width of floating images to 400px in medium screen media query
  • Override the link color and make text #212121
  • Update Lightbox image css to max-width: 50% and max-height: 75%
  • Decrease copyright and email text in mobile to 10px
