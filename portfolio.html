Overview:

I created a portfolio website to showcase my design skills to employers and clients who would like to get to know me better. This website was designed and built with HTML, CSS, and Javascript.

Purpose:

The purpose of the project was to learn how to build and design a website for anyone interested in my designs or is looking for a designer to hire. This frontend project was part of my CareerFoundry course.

Role & Duration:

4 months

  • UI/UX Designer
  • Frontend Developer

Tools and Languages:

  • Atom
  • Adobe Illustrator
  • HTML
  • CSS
  • Javascript

Objective:

Problem -  I was looking to create a portfolio website that would allow my designs to be the main focus. This website provides a way for anyone interested in my designs to view my projects, resume, and learn about me as a person and designer in one place.

Solution - I aimed for my website to be minimal and easy to use. Creating a portfolio website with a minimal design would make it easier for employers and clients to navigate through my website to get to know me and my design abilities.

Process:

First, I learned how to use HTML and CSS to structure my website and design a layout for my portfolio. I designed the website with a mobile-first approach.

At first, the home page had a basic layout with a project image on the left and a text description on the right.

When I started to learn advanced CSS, I created an animation of an image that was created by someone on Freepik. I chose to animate that specific image to hint that I reside in Austin Texas.

I used Sketch to create mid-fidelity wireframes, which helped me determine the visual inconsistencies between my iOS and Android platforms.

Finally, when I started to learn Javascript I was able to make the website more responsive.

I implemented CSS grids to organize my project images in columns and rows on the home page to create a minimal design. Users would be able to read a short description of the project and its process when they hovered over a project image.
