Overview:

Carver Jewels is a shopping app for everyday fine jewelry intended for people who enjoy simple luxuries or buying gifts for someone special in their life. The design process consisted of a mobile-first approach that transitioned to responsive designs, user testing, and a mobile prototype.

Purpose:

The purpose of this project was to create designs for a shopping app based on the branding I established in the beginning. Research shows that fewer people are downloading apps, and I find that especially true for stores. I knew that I wanted to design my app as a responsive web app that would be scalable in larger screen sizes. This was my final project for my CareerFoundry course.

Role & Duration:

1 week

  • UI Designer
  • UX Designer

Tools:

  • Sketch
  • Marvel App
  • Adobe Photoshop

Objective:

Problem -  Some shopping apps can have the problem of being confusing or cluttered, which makes users want to leave the store.

Solution - My goal was to design a shopping app that is inviting, easy to use, and user-centered. I aimed for the design to be clean, bright, and minimal. That would allow users to focus on their tasks or product details. Some key features I included was allowing users to access inventory without an account, having sale items viewable for users to save money, and including a variety of payment options.

Process:

From the beginning, I established brand guidelines to utilize throughout the project to keep the product consistent.


I created a user flow for 3 user stories.

  • User Story 1
    As a new customer, I want to access the inventory without having to register, so that I can make sure this store has what I’m looking for before having to create an account.

  • User Story 2
    As a customer, I want to be able to see when stock is on sale, so that I can save money.

  • User Story 3
    As a customer, I want there to be a variety of payment options, so that I can choose the payment method that suits me best.

Low to mid-fidelity wireframes were designed based on my third user flow where users can view their different payment options. That user flow was chosen because it included user flows 1 and 2 to make the app feel more complete.


User tests were completed and I gained valuable feedback about how the user flow could be improved. I discovered another way users prefer to view payment options by accessing their account information.

Scenario - You are a new user shopping for jewelry and would like to view your payment options. Which steps would you take to accomplish this task?

Feedback from the user test was implemented in my high-fidelity wireframes.

I was challenged to figure out how to make my app stand out. I researched some well-known jewelry stores and found that one based in Austin gives back to women and children’s health and education. That inspired me to add a screen titled Our Cause to show how a purchase can help children in the foster care system. I thought it was a nice addition to the design as an incentive to buy a product because it can help a good cause.

Finally, I created designs for remaining responsive screens for larger devices. Since my app was designed as a responsive web app, scaling the designs to a larger size was made easier.
